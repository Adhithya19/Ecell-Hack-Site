A deep dive into the mechanics of CloudBridge — how we parse, map, analyze, and generate everything a migration team needs to move fast and confidently.

The Process
Four steps,
end to end.
01
Input & Parsing

Ingest Your GCP Infrastructure

Upload your existing GCP configuration in any format — Terraform HCL, Deployment Manager YAML, raw JSON export, or simply describe your setup in natural language. Our parser extracts every resource, dependency, and configuration parameter.

Terraform HCL parser Deployment Manager YAML JSON structured input Natural language input
02
AI Mapping

Intelligent Service Translation

Amazon Bedrock reasons over your infrastructure using a curated GCP→AWS service mapping dataset stored in a FAISS vector database. It selects the closest AWS equivalent for each service, accounts for configuration nuances, and explains every mapping decision with confidence scores.

Amazon Bedrock (reasoning) FAISS semantic retrieval Confidence scoring Mapping explanations
03
Risk & Cost Analysis

Identify Gaps Before They Hurt

A hybrid rule-based and LLM-powered analysis layer scans every mapped service for IAM incompatibilities, unsupported configurations, networking topology mismatches, and data residency issues. Simultaneously, the AWS Pricing API generates a live cost comparison between your GCP baseline and the proposed AWS architecture.

IAM compatibility check Networking gap detection AWS Pricing API Cost delta report
04
Runbook Generation

An Executable Migration Plan

CloudBridge generates a prioritized, dependency-ordered migration runbook with step-by-step instructions, rollback procedures, and validation checkpoints. Output includes AWS CloudFormation templates and Terraform configurations — ready to deploy, not just read.

Dependency ordering CloudFormation output Terraform output Rollback procedures

// CLOUDBRIDGE MIGRATION RUNBOOK — Project: acme-prod — Generated: 2026-03-29
STEP 01
Pre-migration: IAM Role Re-creation
Create equivalent AWS IAM roles for each GCP service account. Map Workload Identity bindings to IRSA (IAM Roles for Service Accounts). Validate least-privilege policies match intended GCP primitives.
PRIORITY: Critical EST: 4 hrs Rollback: N/A (additive)
STEP 02
Network: VPC + Subnet Setup
Provision AWS VPC with equivalent CIDR blocks. Configure subnets to match GCP regional topology. Set up security groups mirroring GCP firewall rules. Enable VPC Flow Logs for audit parity.
PRIORITY: High EST: 2 hrs Rollback: Delete VPC stack
STEP 03
Storage: S3 Bucket Provisioning + Data Sync
Create S3 buckets with equivalent access policies. Enable Cross-Region Replication. Use AWS DataSync or S3 Transfer Acceleration to migrate data from GCS. Validate checksums post-transfer.
PRIORITY: High EST: 6–24 hrs (data-dependent) Rollback: Keep GCS active in parallel
STEP 04
Compute: EC2 Launch + Load Balancer Configuration
Launch EC2 instances from AMIs matching your container/OS baseline. Configure Auto Scaling groups. Set up Application Load Balancer with target groups. Validate health checks before traffic cutover.
PRIORITY: High EST: 3 hrs Rollback: Route53 failback to GCP
